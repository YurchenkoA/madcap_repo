Job Chaining

Job Chaining allows you to link jobs so that they run one directly after another. For example, you can set up a VM backup job, which saves backups locally and then starts a Backup Copy job, which copies the newly created backups to Amazon cloud.


You can link any type of jobs together – backup, backup copy, replication and recovery – and add any number of jobs to the chain. For instance, you can set up a series of backup jobs that trigger one another in the order of priority, or set up a series of Backup Copy jobs, which first send weekly backups to a DR repository and then send monthly backups to Amazon cloud for archiving.
